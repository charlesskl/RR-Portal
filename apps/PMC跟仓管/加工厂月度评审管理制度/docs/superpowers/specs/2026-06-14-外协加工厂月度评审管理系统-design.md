# 外协加工厂月度评审管理系统 — 设计文档

**日期：** 2026-06-14
**状态：** 已批准
**技术栈：** Vue3 + TypeScript + Vite + PocketBase

---

## 1. 项目背景与目标

管理约 200+ 家外协加工厂（注塑/喷油/装配/毛绒车缝四类工艺），实现：
- **数据权责分工**：每个字段明确录入人/审核人，系统强制锁权，不靠人工自觉
- **月度评审流程**：从评分录入 → 自动算分 → 经理审批 → 状态更新的完整线上流程
- **岗位 KPI 自动统计**：记录各岗位关键动作时间+完整度，月度评审时直接看

**规模：** 20–30 个账号，200+ 家工厂
**部署：** 云端（国内 VPS，推荐阿里云/腾讯云 2核4G，Nginx 反代 + HTTPS）

---

## 2. 角色与权限矩阵

| 角色 | 标识 | 可见工厂范围 | 核心权限 |
|------|------|------------|---------|
| 管理员 | admin | 全部 | 全权（含用户管理、评分模板配置） |
| 供应链经理 | sc_manager | 全部 | 审批合作状态、终审红黄牌、月度评审审批 |
| 采购专员-注塑 | buyer_injection | 仅注塑工厂 | 录工厂档案（除资质）、提报合作状态、填资质/交期/整改评分项 |
| 采购专员-喷油 | buyer_painting | 仅喷油工厂 | 同上 |
| 采购专员-装配 | buyer_assembly | 仅装配工厂 | 同上 |
| 采购专员-毛绒车缝 | buyer_sewing | 仅毛绒车缝工厂 | 同上 |
| 财务成本会计 | finance_cost | 全部（只读除产值） | 录入/修改月度产值、年度累计（唯一有写权限） |
| 财务主管 | finance_mgr | 全部（只读） | 审核产值数据 |
| 品质 QC | quality_qc | 全部 | 录入验货报告、巡检记录、不良率等品质评分项、出红黄牌依据 |
| 供应链文员 | sc_clerk | 全部（只读） | 录入品类汇总大盘（表3），系统自动汇总后人工复核 |

**关键锁定规则：**
- `monthly_output`（产值）：仅 `finance_cost` 可写，其他角色只读
- 工厂资质字段：任何人不得随意修改，变更需 admin 或 sc_manager 操作
- 合作状态变更：采购专员仅可"提报"，sc_manager 审批后才正式生效
- 采购专员登录后，工厂列表自动过滤为本工艺品类

## 3. 数据模型（PocketBase 集合）

### 3.1 `users`（PocketBase 内置，扩展字段）
```
role: string        // 角色标识（见上表）
craft: string       // 工艺品类（仅采购专员有值：injection/painting/assembly/sewing）
display_name: string
```

### 3.2 `factories`（工厂档案 — 表1 核心）
```
name: string                   // 工厂名称
craft: string                  // 工艺分类（injection/painting/assembly/sewing）
contact_person: string         // 联系人
contact_phone: string
address: string
workshop_area: number          // 厂房面积(㎡)
equipment_list: json           // 主要设备列表
qualification_files: file[]    // 资质证件附件（受保护字段）
qualification_expiry: date     // 资质有效期
status: string                 // 合作状态：active/limited/suspended/eliminated
status_pending: string         // 待审批的状态变更（采购提报）
status_updated_by: relation(users)
status_updated_at: datetime
created_by: relation(users)
```

### 3.3 `monthly_output`（月度产值 — 仅财务写）
```
factory: relation(factories)
year_month: string             // 格式 "2026-05"
monthly_amount: number         // 当月加工产值（元）
ytd_amount: number             // 年度累计产值（元）
source_doc: string             // 数据来源（结算对账单编号）
entered_by: relation(users)    // 必须是 finance_cost
reviewed_by: relation(users)   // finance_mgr 审核
entered_at: datetime
reviewed_at: datetime
```

### 3.4 `incidents`（异常/事故记录）
```
factory: relation(factories)
incident_date: date
incident_type: string          // batch_defect/env_violation/shutdown/other
description: text
photos: file[]                 // 现场照片
docs: file[]                   // 异常单附件
entered_by: relation(users)    // 采购+品质均可
reviewed_by: relation(users)   // sc_manager
status: string                 // open/closed
close_date: date
```

### 3.5 `score_templates`（评分项模板 — 可配置）
```
name: string                   // 评分项名称（如"交期准时率"）
module: string                 // 所属模块（qualification/delivery/cooperation/defect_rate/process/5s/craft_specific）
max_score: number              // 该项满分
scoring_role: string           // 打分主体（buyer/quality_qc）
craft_filter: string           // 适用工艺（空=通用，否则指定工艺）
is_active: bool
sort_order: number
description: text              // 打分说明/标准
```

**默认通用 70 分模板（初始数据，可在后台改）：**
| 模块 | 评分项 | 满分 | 打分主体 |
|------|--------|------|---------|
| 资质合规 | 证件齐全有效 | 10 | 采购专员 |
| 交期管理 | 交期准时率 | 20 | 采购专员 |
| 配合度 | 整改响应配合度 | 10 | 采购专员 |
| 不良率 | 月度综合不良率 | 15 | 品质 QC |
| 制程 | 制程检验通过率 | 10 | 品质 QC |
| 现场 5S | 5S 现场评分 | 5 | 品质 QC |

**工艺专项 30 分模板**（各工艺各配，admin 在后台设定）

### 3.6 `monthly_scores`（月度评分 — 表2）
```
factory: relation(factories)
year_month: string
score_items: json              // [{template_id, score, notes}] 逐项得分
total_score: number            // 系统自动算（公式锁定）
grade: string                  // A/B/C/D（系统自动判定）
flag: string                   // none/yellow/red
flag_reason: text
flag_issued_by: relation(users) // 品质出依据
flag_approved_by: relation(users) // sc_manager 终审
correction_plan: text          // 整改计划
correction_due: date
correction_closed: bool
status: string                 // draft/submitted/approved
submitted_by: relation(users)
approved_by: relation(users)
```

**自动评级规则（默认，admin 可改）：**
- ≥ 90 → A 级（正常）
- 80–89 → B 级（正常）
- 70–79 → C 级（限单）
- < 70 → D 级（暂停/淘汰评审）

**红黄牌默认触发：**
- 红牌：批量不良事故 / 环保违规 / 停工 → 强制进暂停或淘汰评审
- 黄牌：单次重大异常 / 连续 2 月分数下滑 → 限单 + 整改复查

### 3.7 `review_meetings`（月度评审会议 — 表3）
```
year_month: string
summary_by_craft: json         // 各品类工厂数量、等级分布、平均得分、总产值
optimization_suggestions: text // 供应链优化/整合/淘汰建议
participants: json             // 参会人员
meeting_date: date
approved_by: relation(users)   // sc_manager
status: string                 // draft/approved
```

### 3.8 `kpi_logs`（岗位履职记录 — 自动生成）
```
user: relation(users)
action_type: string            // output_entered/output_reviewed/score_submitted/summary_done/correction_closed/...
target_month: string           // 针对哪个月的数据
deadline: datetime             // 该动作的 SLA 截止时间
completed_at: datetime         // 实际完成时间
is_on_time: bool               // 系统自动算
notes: text
```

## 4. 评分引擎

- 总分由系统按 `score_templates` 加权求和，前端公式锁定，文员不能手动改总分
- 等级判定在提交评分时自动计算并写入 `grade` 字段
- 红黄牌由品质 QC 在评分单填写依据 → sc_manager 在评审会审批生效
- 评分单状态：草稿（draft）→ 采购+品质各自提交（submitted）→ sc_manager 审批（approved）

---

## 5. 月度评审标准流程

```
每月1日-4日：采购/品质完成上月评分录入（draft → submitted）
每月5日：财务成本会计录入上月产值 → 财务主管审核
每月6日：供应链文员确认品类汇总大盘自动汇总数据
每月7-8日：月度评审会议
  - sc_manager 查看大盘+评分+红黄牌
  - 审批合作状态变更（正常/限单/暂停/淘汰）
  - 形成整改计划下发
  - approved → 工厂合作状态统一更新
```

系统在每月 1 日自动生成当月评分单草稿（基于上月工厂列表），并向各角色发送待办提醒（站内消息）。

## 6. 岗位 KPI 看板

系统自动从 `kpi_logs` 聚合以下指标，月度评审时展示：

| 岗位 | KPI 指标 | SLA 截止 |
|------|---------|---------|
| 财务成本会计 | 产值录入按时率 | 每月5日 |
| 财务主管 | 产值审核按时率 | 每月5日 |
| 采购专员 | 评分录入完整率、整改闭环率 | 每月4日 |
| 品质 QC | 验货报告及时率、评分录入完整率 | 每月4日 |
| 供应链文员 | 汇总大盘完成按时率 | 每月6日 |

## 8. 云端部署方案

**推荐：国内 VPS（阿里云/腾讯云 2核4G）**

```
服务器结构：
  /opt/factory-review/
    pocketbase          # 单二进制，监听 8090
    pb_data/            # 数据库+附件
    dist/               # 前端打包静态文件

Nginx 配置：
  - / → 托管 dist/ 静态文件
  - /api/ → 反代 PocketBase 8090
  - HTTPS：Let's Encrypt 自动续签

备份：
  - PocketBase 内置 backup API，每日定时备份
  - 备份文件上传阿里云 OSS / 腾讯云 COS
```

**本地开发：** 桌面 `加工厂月度评审管理制度/` 文件夹，`bin/pocketbase serve` + `npm run dev`，局域网内可访问。
