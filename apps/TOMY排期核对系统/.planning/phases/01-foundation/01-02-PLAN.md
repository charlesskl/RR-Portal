<task type="auto">
  <name>Task 1: Create Dockerfile, docker-compose.yml, and nginx.conf for production deployment</name>
  <files>Dockerfile, docker-compose.yml, nginx/nginx.conf</files>
  <action>
1. Create `nginx/nginx.conf` (the nginx/ directory may already exist but the conf may need overwriting):
   ```nginx
   server {
       listen 80;
       client_max_body_size 50M;  # MUST be at server block level, NOT inside a location

       location / {
           root /usr/share/nginx/html;
           try_files $uri $uri/ /index.html;
       }

       location /api/ {
           proxy_pass http://app:3000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_read_timeout 120s;
       }
   }
   ```
   CRITICAL: `client_max_body_size 50M` MUST be at the `server` block level. Putting it in a `location` block causes new routes in later phases to silently get the 1MB default.

2. Create `Dockerfile` with multi-stage build:
   - Stage 1 (`frontend-build`): FROM node:24-alpine, WORKDIR /app/client, COPY client/package*.json, RUN npm ci, COPY client/, RUN npm run build
   - Stage 2 (`backend`): FROM node:24-alpine, WORKDIR /app, COPY package*.json, RUN npm ci --production, COPY server/ ./server/, COPY --from=frontend-build /app/client/dist ./client/dist, EXPOSE 3000, CMD ["npx", "tsx", "server/index.ts"]

   IMPORTANT: tsx is a production dependency (moved to `dependencies` in Plan 01-01 per user decision). `npm ci --production` WILL install it. Do NOT add a separate `RUN npm install -g tsx` step -- it is already available via the production install. Verify by confirming `tsx` appears under `dependencies` (not `devDependencies`) in package.json.

3. Create `docker-compose.yml`:
   ```yaml
   version: '3.9'
   services:
     app:
       build: .
       expose:
         - "3000"
       environment:
         - NODE_ENV=production
       healthcheck:
         test: ["CMD", "wget", "-qO-", "http://localhost:3000/health"]
         interval: 10s
         timeout: 5s
         retries: 3

     nginx:
       image: nginx:alpine
       ports:
         - "80:80"
       volumes:
         - ./nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro
       depends_on:
         app:
           condition: service_healthy
   ```

4. Verify nginx.conf contains `client_max_body_size 50M` at server level (not nested in location).
