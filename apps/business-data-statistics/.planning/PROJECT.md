# 三工结余系统 — BUG修复与体验升级

## What This Is

工厂生产经营数据统计系统（三工结余模块），服务于统计组和各部门录入员。管理啤机部、印喷部、装配部三个生产部门的日常产值、人数、费用数据，自动计算结余。系统已完成基础功能开发，处于测试阶段，需要修复已知缺陷并提升用户体验后才能正式投入使用。

## Core Value

录入员能高效、准确地录入每日生产数据，统计组能快速获取可靠的汇总报表交给管理层。

## Context

- 技术栈：Express 5 + PostgreSQL 17 + Vue 3 + Element Plus（CDN）+ SheetJS
- 前端是 2000+ 行单文件 `app.js`，无构建步骤
- 后端配置驱动：`modules/balance/config.js` 是字段定义的唯一真相来源
- 系统最高频 BUG 类型是前后端字段名不一致
- 公司内网环境，Chrome 浏览器，数据量约每年 15000 条
- 用户是编程新手，所有技术决策由 Claude 负责实现

## Constraints

- **Tech stack**: 保持现有技术栈不变（Express + PostgreSQL + Vue 3 CDN），不引入构建工具
- **Compatibility**: 公式设置窗口的计算规则需要安全执行，不能用 eval 直接执行用户输入
- **Data safety**: 修复 BUG 不能影响现有测试数据（print_records 已有 18 条）
- **Single file**: 前端 app.js 单文件架构维持现状，不做拆分
