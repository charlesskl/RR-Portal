   将 `prop="workshop"` 改为 `prop="workshop_name"`。
   原因：后端 records.js GET 查询的 SQL 用 `w.name as workshop_name` 返回车间名，前端 prop 必须与 SQL AS 别名一致。

2. **BUG-04 -- lockForm 字段名（约第1375行及其模板引用）：**
   - 将 lockForm 数据定义中的 `dept: ''` 改为 `department: ''`
   - 在 template 中搜索所有 `lockForm.dept` 引用，全部改为 `lockForm.department`
   - 包括 `v-model="lockForm.dept"` 改为 `v-model="lockForm.department"`
   - 包括提交函数中可能的 `lockForm.dept` 引用
   原因：后端 settings.js POST /data-locks 解构 `const { department, lock_month } = req.body`，前端必须发送 `department` 而非 `dept`。

3. **BUG-07 -- 锁定人列 prop（约第1338行）：**
   将 `prop="locked_by"` 改为 `prop="locked_by_name"`。
   原因：后端 settings.js GET /data-locks 已 JOIN users 表返回 `u.name as locked_by_name`，前端需匹配此别名。

**不要修改上述三处以外的任何代码。**
