**ENTRY-02 Tab 键导航实现：**

1. 在 DeptRecordsPage 的 methods 中新增 `handleTabKey(row, currentField, event)` 方法:

```javascript
handleTabKey(row, currentField, event) {
  // 1. 获取当前单元格的值（用于传给 saveCell 的模拟 event）
  const value = event.target.value;

  // 2. 先保存当前单元格
  //    saveCell 第一行就重置 editingCell，所以这里直接调用
  //    构造一个简单的伪 event 传给 saveCell（saveCell 只读 event.target.value）
  this.saveCell(row, currentField, event);

  // 3. 在 editableColumns 中找当前字段的位置
  const editableCols = this.editableColumns; // computed 属性
  const currentIdx = editableCols.findIndex(c => c.field === currentField);

  // 4. 判断 Shift+Tab（反向）还是 Tab（正向）
  if (event.shiftKey) {
    // Shift+Tab: 跳到上一个可编辑字段
    const prev = currentIdx > 0 ? editableCols[currentIdx - 1] : null;
    if (prev) {
      this.$nextTick(() => this.startEdit(row, prev));
    }
    // 如果已是第一列，不做任何事（取消编辑状态，saveCell 已处理）
  } else {
    // Tab: 跳到下一个可编辑字段
    const next = currentIdx < editableCols.length - 1 ? editableCols[currentIdx + 1] : null;
    if (next) {
      this.$nextTick(() => this.startEdit(row, next));
    }
    // 如果已是最后一列，不做任何事（取消编辑状态，saveCell 已处理）
  }
}
```

2. 修改行内 input 模板（app.js 第 337-342 行区域），在现有 `<input>` 标签上新增一行事件绑定:

将:
```html
<input :value="row[col.field]" @blur="saveCell(row, col.field, $event)"
       @keyup.enter="$event.target.blur()"
       @keyup.escape="cancelEdit"
       @input="limitDecimals($event)"
```

改为:
```html
<input :value="row[col.field]" @blur="saveCell(row, col.field, $event)"
       @keyup.enter="$event.target.blur()"
       @keyup.escape="cancelEdit"
       @keydown.tab.prevent="handleTabKey(row, col.field, $event)"
       @input="limitDecimals($event)"
       autofocus />
```

注意事项:
- 必须是 `@keydown.tab.prevent`（不是 @keyup），否则浏览器在 keydown 阶段就已移动焦点
- `.prevent` 修饰符阻止浏览器默认 Tab 行为
- handleTabKey 中调用 saveCell 后 editingCell 已被重置为 null，$nextTick 中的 startEdit 会重新设置
- 支持 Shift+Tab 反向跳转（event.shiftKey 检测）
