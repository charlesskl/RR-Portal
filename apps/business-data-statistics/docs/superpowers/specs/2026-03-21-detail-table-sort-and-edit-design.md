# 设计文档：明细表排序固定与编辑增强

**日期：** 2026-03-21
**范围：** 三个部门明细表（啤机部 / 印喷部 / 装配部）统一适用
**状态：** 待实现

---

## 背景与问题

当前明细表存在以下问题：
1. 编辑任意数字字段后，该行会跳到同日期同车间组的最顶部（原因：SQL 按 `updated_at DESC` 排序，改动后时间戳更新导致排位变化）
2. 新增行插入位置不固定，日期排列无规律
3. 存在序号列，占用空间且无实际业务意义
4. 日期和车间字段为纯文本，无法在行内编辑，录入员必须删行重建

---

## 需求列表

| ID | 描述 |
|----|------|
| SORT-01 | 明细表按日期升序排列（1日 → 2日 → 31日） |
| SORT-02 | 同日期内按车间固定顺序排列 |
| SORT-03 | 同日期同车间内按 id 升序排列（创建先后，编辑不影响顺序） |
| SORT-04 | 编辑任意字段后，行位置不发生变化（除非编辑的是日期或车间本身） |
| ADD-01 | 新增行后立即按上述规则插入正确位置（不停留在末尾） |
| DEL-01 | 去掉序号列，三个部门明细表统一 |
| EDIT-01 | 日期列支持行内编辑（日期选择器），保存后立即重排 |
| EDIT-02 | 车间列支持行内编辑（下拉选择器），保存后立即重排 |
| EDIT-03 | 日期和车间均为必填，不允许清空保存 |

## 车间固定顺序

每个部门的车间排序权重（rank 值越小越靠前）：

| 车间 | 啤机部 rank | 印喷部 rank | 装配部 rank |
|------|------------|------------|------------|
| 兴信A | 1 | 1 | 1 |
| 兴信B | 2 | — | 2 |
| 华登A | 3 | 2 | 3 |
| 邵阳华登 | 4 | 3 | 4 |
| 华嘉 | — | — | 5 |

## 设计详情

### 1. 排序逻辑

**后端（SQL）**

所有三个部门的记录查询（`GET /api/:dept/records`）改为三级 ORDER BY：

```sql
ORDER BY
  date ASC,
  CASE workshop_name
    WHEN '兴信A' THEN 1
    WHEN '兴信B' THEN 2
    WHEN '华登A' THEN 3
    WHEN '邵阳华登' THEN 4
    WHEN '华嘉' THEN 5
    ELSE 99
  END ASC,
  id ASC
```

> 注：车间名称从 `workshops` 表 JOIN 获取，CASE WHEN 基于 `name` 字段。
> 华嘉仅装配部有，其余部门的 CASE 不含该项（或排 99 不影响结果）。

**前端（Vue）**

在 `app.js` 中定义一个 `sortRecords(records, dept)` 工具函数，使用相同的三级排序逻辑（date → workshop rank → id）。

在以下操作完成后立即调用 `sortRecords` 重排本地 `records` 数组：
- `loadRecords()`（数据加载完成后）
- `saveCell()`（保存单元格后）
- `addRow()`（新增行后）
- `editDate()`（日期保存后）
- `editWorkshop()`（车间保存后）

这样无论是后端返回的数据还是本地操作，显示顺序始终一致，不依赖 DOM 刷新。

---

### 2. 去掉序号列

从三个部门明细表的 `<el-table-column>` 定义中删除序号列（通常为 `type="index"` 或手动渲染的序号列）。

不需要任何后端改动，无数据迁移风险。

---

### 3. 日期与车间行内编辑

**当前状态：** 日期和车间在行创建时写入，之后以纯文本显示，无法修改。

**目标状态：** 与其他可编辑字段相同，点击后进入编辑态。

#### 日期列（EDIT-01）

- 点击日期单元格 → 显示 `<el-date-picker>` 日期选择器
- 选择新日期后，调用 `PUT /api/:dept/records/:id`，`body` 包含 `{ record_date: '2026-03-10' }`
- 保存成功后，本地更新该行 `record_date` 字段，调用 `sortRecords` 重排，行移动到新日期对应位置
- 若清空日期直接关闭，不保存，恢复原值（前端校验）

#### 车间列（EDIT-02）

- 点击车间单元格 → 显示 `<el-select>` 下拉，选项为该部门所有车间（与新增行下拉一致）
- 选择后调用 `PUT /api/:dept/records/:id`，`body` 包含 `{ workshop_id: 3 }`
- 保存成功后更新本地 `workshop_name` 和 `workshop_id`，调用 `sortRecords` 重排
- 若关闭未选择，恢复原值（前端校验）

#### 必填校验（EDIT-03）

- 日期和车间的编辑器关闭时，若值为空则不保存，弹出提示："日期为必填项，不能清空"/"车间为必填项，不能清空"

## 测试验收标准

1. 编辑啤机部某行的任意数字字段，该行在表格中的位置不发生变化
2. 新增一行日期为"15日"，行立即出现在14日行之后、16日行之前
3. 明细表中不显示序号列
4. 点击某行日期，出现日期选择器，改为"20日"后行移动到正确位置
5. 点击某行车间，出现下拉选择器，改为"邵阳华登"后行移动到该日期内邵阳华登对应位置
6. 以上行为在印喷部、装配部明细表中一致
