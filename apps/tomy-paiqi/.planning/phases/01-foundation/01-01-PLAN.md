  <files>package.json, tsconfig.json, server/index.ts, server/routes/upload.ts, .gitignore, vitest.config.ts, eslint.config.js</files>
  <action>
1. Create `.gitignore` with: `node_modules/`, `dist/`, `*.pdf`, `*.xlsx`, `!client/**/*.xlsx`, `DATA/`, `.env`, `*.log`, `client/dist/`. This protects the real PO/schedule files from being committed.

2. Run `npm init -y` in project root to create `package.json`.

3. Install backend dependencies (NOTE: tsx MUST be a production dependency, not devDependency, because the Docker image uses `npm ci --production` and needs tsx to run the server):
   ```
   npm install express cors multer pdf-parse pdfjs-dist exceljs chinese-days date-fns tsx
   ```

4. Install dev dependencies:
   ```
   npm install -D typescript @types/node @types/express @types/multer @types/cors @types/pdf-parse vitest eslint @typescript-eslint/parser
   ```

5. Create `tsconfig.json` for the server (target: ES2022, module: NodeNext, moduleResolution: NodeNext, outDir: dist, rootDir: server, strict: true, esModuleInterop: true, resolveJsonModule: true). Include only `server/**/*`.

6. Create `server/index.ts`:
   - Import express, cors
   - Configure `app.use(cors())`, `app.use(express.json({ limit: '50mb' }))`
   - Mount uploadRouter at `/api`
   - GET `/health` returns `{ status: 'healthy' }`
   - Listen on `process.env.PORT || 3000`

7. Create `server/routes/upload.ts`:
   - Import multer, configure MemoryStorage with `limits: { fileSize: 25 * 1024 * 1024 }`
   - Export `uploadRouter` (express.Router)
   - POST `/process` accepts multipart files (fields: `pos` multiple, `schedule` single), returns `{ status: 'ok', message: 'Phase 1 stub - not yet implemented' }`

8. Add scripts to package.json: `"dev": "npx tsx server/index.ts"`, `"build": "npx tsc"`, `"test": "npx vitest run"`.

9. Create minimal `vitest.config.ts` pointing to `server/` directory.

10. Create minimal `eslint.config.js` with TypeScript parser config.

IMPORTANT: Use multer v2.1.1 API (not v1 API). The v2 API uses `multer({ storage: multer.memoryStorage(), limits: { fileSize: 25 * 1024 * 1024 } })`. Do NOT install xlsx or sheetjs packages.

IMPORTANT: tsx MUST be in `dependencies` (not `devDependencies`) so it is available in the Docker production image. This is a deliberate choice for this internal tool -- tsx is the runtime, not just a dev convenience.
