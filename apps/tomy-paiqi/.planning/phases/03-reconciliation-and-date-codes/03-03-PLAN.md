  result: ReconciliationResult,

Implementation:
1. Load scheduleBuffer into ExcelJS workbook. Get worksheet '总排期'.
2. Build column map from header row (same pattern as excelParser.ts — `buildColumnMap`).
3. Resolve column indices for all comparison fields + 日期码 column.
4. Add status column at `ws.columnCount + 1`, set header cell to "状态".
5. For each RowMatchResult with status 'matched':
   a. Get the row by scheduleRowIndex
   b. For each mismatch in mismatches: find the column index for mismatch.field, set cell.fill to light red `{ type: 'pattern', pattern: 'solid', fgColor: { argb: 'FFFFCCCC' } }`, set cell.note to `PO value: ${mismatch.poValue}`
   c. For non-mismatched cells on matched rows: set green fill `{ argb: 'FF90EE90' }` (apply to all data cells in the row, not just comparison fields)
   d. Set status column cell to "已核对"
   e. If dateCode is not null AND the 日期码 cell is empty (null/undefined/''): write dateCode to that cell
6. Track matched schedule row indices. For all schedule rows NOT in the matched set but sharing the same tomyPO as a matched row: set status to "未核对" (no green fill).
7. For each unmatchedPOItem: append a new row at the bottom of the sheet with available PO data, apply yellow fill `{ argb: 'FFFFFF99' }` to all cells in the row, set status to "未匹配".
8. Return `workbook.xlsx.writeBuffer() as Promise<Buffer>`.

Column field-name mapping (field name in FieldMismatch.field to column header in schedule):
- Use the same COLUMN_ALIASES pattern from excelParser.ts for tomyPO and customerPO
- Direct column names: 国家, 第三客户名称, 客跟单, 货号, 数量, 外箱, PO走货期, 箱唛资料, 日期码
