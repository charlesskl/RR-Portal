# TOMY排期核对系统

## What This Is

一个网页应用，用于将TOMY的PDF格式PO（采购订单）与排期Excel表进行自动核对。系统能解析PDF中的PO数据，与排期表逐字段比对，不一致的内容标红高亮，并根据PO编号中的工厂代码（RR01=东莞，RR02=印尼）自动将订单分类到对应文件夹。同时可根据PO走货期自动生成并填入日期码。

## Core Value

准确核对PO与排期表数据，快速发现不一致项并标红提示，减少人工核对的时间和出错率。

### Active

- [ ] 逐字段核对PO与排期表：接单期国家、第三客户名称、客跟单、TOMY PO、CUSTOMER PO、货号、数量、外箱、总箱数、PO走货期、箱唛资料

- [ ] 东莞和印尼的核对结果分别保存到两个文件夹中
- [ ] 自动生成日期码并填入排期表（格式：月字母+日+年+工厂代码，如D1526RR02）

- [ ] 月份编码规则：A=1月, B=2月, C=3月, D=4月, E=5月, F=6月, G=7月, H=8月, I=9月, J=10月, K=11月, L=12月

### Out of Scope

- 扫描件/图片PDF的OCR识别 — 当前PO均为可复制文字的PDF
- ERP系统集成 — 仅处理文件级别的核对
- 移动端适配 — 以桌面浏览器为主

## Context

- TOMY是玩具品牌，有东莞和印尼两个生产基地
- PO文件为PDF格式，内容可复制文字，非扫描件
- 排期表为已有的Excel模板，有固定的列名和格式（用户将提供模板）
- PO编号中包含工厂代码：RR01代表东莞工厂，RR02代表印尼工厂
- 日期码示例：D1526RR02 表示2026年4月15日、印尼工厂
- 日期码计算：走货期往前推1个月，如果落在周末或节假日则继续往前推到最近的工作日
- 需要支持中国节假日日历（影响工作日判断）

## Constraints

- **Tech stack**: 网页应用，需能在不同电脑上使用
- **数据格式**: PDF输入（可复制文字）+ Excel输入/输出
- **模板依赖**: 排期表格式依赖用户提供的已有Excel模板
