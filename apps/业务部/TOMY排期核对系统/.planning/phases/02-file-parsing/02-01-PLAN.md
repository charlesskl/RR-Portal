  数量: number | null

Create `server/lib/pdfExtractor.ts`:

1. **pdfjs-dist setup** (CRITICAL — must use legacy build on Node.js):
   ```typescript
   import { getDocument, GlobalWorkerOptions } from 'pdfjs-dist/legacy/build/pdf.mjs'
   import { pathToFileURL, fileURLToPath } from 'url'
   import { dirname, resolve } from 'path'
   ```
   Set `GlobalWorkerOptions.workerSrc` using `pathToFileURL()` (Windows requires file:// URL).
   Set `standardFontDataUrl` to suppress warnings.

2. **extractPDFText(buffer)**: Iterate all pages, concatenate text content items with newlines.

3. **extractPO(buffer, filename)**: Main export function.
   - Call extractPDFText to get raw text
   - Extract header fields using label-anchored regex (from research Pattern 1):
     - `Purchase Order No\s*:\s*(\d+)` for tomyPO
     - `Customer PO No\s*:\s*(\S+)` for customerPO
     - `Handle By\s*:\s*([^\n(]+)` for handleBy
     - `Customer Name:\s*\n?\s*([^\n]+)` for customerName
     - `Port of Discharge \/ Destination Country:\s*\n?\s*([^\n]+)` for destCountry
   - Extract items using multi-item pattern (from research Pattern 2):
     - Find Part No. lines followed by revision code, date, quantity
     - For each item, extract factory code (RR01/RR02) from nearby text
     - Extract carton packing from "N EA/SET / MASTER CARTON" pattern
   - Return POData with all fields populated

4. **IMPORTANT edge cases from research:**
   - PDF header repeats on every page — only extract header fields once
   - Part numbers can contain letters and numbers (47280A, T72465ML3, E73856)
   - Factory code appears on its own line after the product description
   - Some POs have 1 item, others have 2-4 items
   - The text stream has blank lines with single spaces between label and value
