**Step 2 — Update upload.ts route** (`server/routes/upload.ts`):

Add imports at top:
```typescript
import { buildZipBuffer } from '../lib/zipBuilder.js'
import { buildSummaryReport } from '../lib/summaryReport.js'
```

Change multer fields from `schedule` to dual fields:
```typescript
upload.fields([
  { name: 'pos', maxCount: 20 },
  { name: 'scheduleDg', maxCount: 1 },
  { name: 'scheduleId', maxCount: 1 },
])
```

Replace the single-schedule processing block with dual-schedule processing:

1. Read `files.scheduleDg` and `files.scheduleId` separately (both optional)
2. For each present schedule file:
   - Parse with `parseScheduleExcel(buffer)`
   - Store the buffer and parsed rows
   - Set `response.scheduleDg` or `response.scheduleId` with filename/status/rowCount
3. Read clean filenames from `req.body.scheduleDgName` and `req.body.scheduleIdName`

Replace the single reconciliation block with dual reconciliation:

1. Build the full `poDataList` from successful PO results (same as before)
2. Filter POs by factory code for each schedule:
   ```typescript
   const dgPOs = poDataList.filter(po => po.items[0]?.factoryCode === 'RR01')
   const idPOs = poDataList.filter(po => po.items[0]?.factoryCode === 'RR02')
   ```
3. If 东莞 schedule buffer exists: `const dgResult = reconcile(dgPOs, dgScheduleRows)` and `const dgExcel = await writeAnnotatedSchedule(dgBuffer, dgResult, dgScheduleRows)`
4. If 印尼 schedule buffer exists: `const idResult = reconcile(idPOs, idScheduleRows)` and `const idExcel = await writeAnnotatedSchedule(idBuffer, idResult, idScheduleRows)`
5. Build summary report: `const summaryText = buildSummaryReport(dgResult ?? emptyResult, idResult ?? emptyResult)` where `emptyResult` is `{ matched: [], unmatchedPOItems: [], ambiguousPOItems: [], errors: [] }`
6. Build ZIP entries array — only include entries for schedules that were uploaded:
   ```typescript
   const zipEntries: Array<{ name: string; buffer: Buffer }> = []
   if (dgExcel) zipEntries.push({ name: 'DG/东莞排期核对结果.xlsx', buffer: dgExcel })
   if (idExcel) zipEntries.push({ name: 'ID/印尼排期核对结果.xlsx', buffer: idExcel })
   zipEntries.push({ name: '核对汇总报告.txt', buffer: Buffer.from(summaryText, 'utf-8') })
   ```
7. If any entries exist, build ZIP and store: `const zipBuffer = await buildZipBuffer(zipEntries)` then `storeOutput(zipBuffer)`
8. Set `response.reconciliationDg` and `response.reconciliationId` with per-factory stats (same shape as old `reconciliation`)
9. Wrap all reconciliation in try/catch — failure is non-fatal (per existing pattern)

Update download endpoint headers:
```typescript
res.setHeader('Content-Type', 'application/zip')
res.setHeader('Content-Disposition', 'attachment; filename="TOMY_reconciliation.zip"')
```

CRITICAL per research Pitfall 3: Filter POs by factory code BEFORE reconciling. Do NOT reconcile all POs against both schedules — that creates spurious unmatched entries. Use `po.items[0]?.factoryCode` since all items in a real PO share the same factory code.
