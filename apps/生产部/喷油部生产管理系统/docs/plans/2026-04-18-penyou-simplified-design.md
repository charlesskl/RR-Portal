# 喷油部系统 · 简化版设计(2026-04-18)

> 本文档替换 `2026-04-16-penyou-system-design.md` 的范围。里程碑 1(骨架)保留;里程碑 2(核价 API + Excel 导入)部分保留;其余(工人/工单/计件/报表)**取消**。

## 一、背景与本次调整

原计划想做「排产 + 计件 + 工资 + 报表」一整套。用户反馈太复杂,实际需要的核心流程只有三步:

1. **录核价**:把核价表数据存进系统(从 Excel 导入 或 网页手录)
2. **分拉**:输入货号 → 系统列出所有位置/工序 → 主管逐行选一个「拉」(宋沛霖手喷 / 宋沛霖自动 / 胡旗移印)
3. **在「收支表」页面看/改/导出**:系统按分拉自动填能算的列,其它列主管在网页上直接编辑,需要时导出 xls 到桌面

以前规划的工人名册、工单、计件录入、工资报表 **全部砍掉**。

## 二、范围

**包含:**

- 核价表数据库 + Excel 导入 + 手工增改删
- 分拉界面(按货号 → 工序列表 → 每行选拉)
- **收支表网页**(类 Excel 网格,Luckysheet 渲染,能算的列自动填,手填列浏览器里编辑并落库)
- 一键导出当前网页快照为 xls 到桌面

**不包含:**

- 工人名册 / 工单管理 / 计件录入 / 工资计算 / 任何工资报表
- 收支表 32 列里看不懂的 5 列(无产值工资 / 可收回工资 / 可收回印尼工资 / 可回收油漆金额 / 加工费)—— 列先摆在那,值留空
- 产值/工资算法的「实际版」(每天录生产数) —— **用计划版**:产值 = Σ 目标数 × 报价,工资 = Σ 目标数 × 工价

## 三、数据模型

### 保留(里程碑 1 已建)

- `products`:货号/货名/报价/备注
- `product_processes`:位置/工艺/目标数/人数/工价/核价/油漆价/总核价

### 新增

#### `lines` 拉
| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | |
| name | TEXT | 拉名(种子数据 3 条:宋沛霖手喷 / 宋沛霖自动 / 胡旗移印) |
| sort_order | INTEGER | 排序,决定收支表里 3 行的顺序 |

#### `dispatches` 分拉记录
| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | |
| dispatch_date | DATE | 分拉日期 |
| product_id | INTEGER FK | |
| product_process_id | INTEGER FK | 具体分哪个工序 |
| line_id | INTEGER FK | 分到哪条拉 |
| created_at | DATETIME | |

唯一约束:`(dispatch_date, product_process_id)` —— 同一天同一道工序只能分给一条拉。

#### `ledger_edits` 收支表手填单元格
| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PK | |
| ledger_date | DATE | 日期(收支表的 A 列) |
| line_id | INTEGER FK | 拉 |
| product_id | INTEGER FK | 货号(可空,某些行可能不绑货号) |
| column_key | TEXT | 收支表列 key(如 `material`, `rent`, `utilities` …) |
| value | TEXT | 单元格值(文本存,前端转数字) |

**为什么稀疏键值表,不是一行 32 列的宽表?**

- 收支表列名可能以后还要加(老板说要看新指标)—— 加列时不用改 schema
- 大部分单元格是空的,稀疏存省空间且查询干净
- 导出时:把这些手填值和自动算的值一合并,写进 xlsx

## 四、页面结构

### 1. `/products` 核价表管理(已做好,保留)

现状:顶部搜索 + 新建 + 导入 Excel,表格含工序数,展开行看工序明细,弹窗编辑。**不动**。

### 2. `/dispatch` 分拉(新增)

```
┌─────────────────────────────────────────┐
│ 日期: [2026-04-18 ▾]                     │
│ 货号: [搜索货号/货名 🔍]                 │
└─────────────────────────────────────────┘

找到货号后:

┌─ 47723 JD 2#农场车套装 ─ 客户报价 1.09 ──┐
│ 位置  工艺   目标数  工价   核价   分到  │
│ 车轮  喷油   1800   0.4    0.688  [宋沛霖手喷▾] │
│ 铲头  喷油   2400   0.075  0.129  [宋沛霖手喷▾] │
│ 车身  移印   5000   0.144  0.248  [胡旗移印▾  ] │
│ ...                                      │
│                       [保存分拉]          │
└─────────────────────────────────────────┘
```

- 选好后点保存 → `POST /api/dispatches` 批量写入(同日期+同货号先删旧的,再写新的)
- 已分过的货号,进来默认回显上次的选择,可改

### 3. `/ledger` 收支表(新增,核心)

**布局:** 上方日期选择器 + 「导出 xls」按钮,下方全屏 Luckysheet 网格。

**网格内容:**

- 表头:按收支表(14).xls 第 2 行 32 列(日期/拉名/机台数/…/结余%/备注)
- 数据行:遍历当天所有已分拉的货号 × 每个货号 3 条拉,= N × 3 行
- **自动填的列**(浅蓝底,只读):
  - A 日期、B 拉名、I 工时(=11)
  - K 总产值 = Σ(该货号分到该拉的位置的 目标数 × 报价)
  - M 员工总工资 = Σ(该货号分到该拉的位置的 目标数 × 工价)
  - J 总时间 = H × I(H 是手填后自动)
  - L 员工人均产值 = K / H
  - E 开机率 = D / C
  - O 总工资占产值% = M / K
  - AE 结余 = K - 所有成本列之和
  - AF 结余% = AE / K
- **手填列**(白底,可编辑):C 机台数 / D 每天开机数 / F 管工人数 / G 杂工 / H 员工人数 / N 管工工资 / P 设备投资 / Q~W 各种成本 / X 补贴 / Y 实际用原料金额 / AG 备注 …
- 用户在格子里改值 → onChange 防抖 500ms → `POST /api/ledger/edits` 落 `ledger_edits` 表
- 5 列看不懂的(Z/AA/AB/AC/AD)正常显示列头,格子当普通手填列处理(可填可不填)

**导出按钮:** 用后端 `exceljs` 生成 xls,把当前网格内容(含自动算出的 + 手填的)直接写成文件,浏览器下载到 `C:\Users\Administrator\Desktop\核价分拉-YYYYMMDD.xls`。

### 4. 导航去掉其它菜单

左侧菜单保留 3 项:核价表 / 分拉 / 收支表。工人/工单/计件/报表 的占位页删除。

## 五、后端路由

### 新增

- `GET /api/lines` — 返回 3 条拉种子数据
- `GET /api/dispatches?date=&product_id=` — 查当日某货号的分拉
- `POST /api/dispatches` body `{date, product_id, items:[{product_process_id, line_id}]}` — 事务里先删旧再写新
- `GET /api/ledger?date=` — 返回当日收支表全部数据(= 自动算 + 手填合并),前端直接喂给 Luckysheet
- `POST /api/ledger/edits` body `{date, line_id, product_id, column_key, value}` — upsert 单个手填单元格
- `GET /api/ledger/export?date=` — 生成 xls,Content-Disposition 下载;文件名 `核价分拉-YYYYMMDD.xls`

### 保留(里程碑 2 已做)

- `GET/POST/PUT/DELETE /api/products` + `POST /api/products/import`

## 六、核价 Excel 导入改进(必须做)

现状:Sheet 1 能导,其余 4 个 sheet 格式不同全跳过。**必须改造**,不然数据不全:

**策略:按每个 sheet 自己的表头识别列**。

1. 读每个 sheet 第 2 行(表头),从单元格文本匹配关键字:
   - `货号` → code 列
   - `货名` → name 列(独立列时)
   - `位置` 或 `工序` 第一次出现 → part_name 列
   - `工艺` 或 `工序` 第二次出现 → technique 列
   - `目标数` / `人数` / `工价` / `核价` / `油漆价` / `总核价` / `报价` / `备注` → 对应列
2. 若 Sheet 1 那种「货号+货名」合在一格(多行文本),保留原有 split 逻辑作为 fallback
3. 若 sheet 里完全没有货号列(如 47600 货柜车),用 sheet 名的数字前缀(`^\d+`)当 code,sheet 名当 name
4. 其它行按识别到的列索引取值,遇到合计/小计行(工序列或位置列为空)跳过

出单测,每个 sheet 至少覆盖一条样例。
