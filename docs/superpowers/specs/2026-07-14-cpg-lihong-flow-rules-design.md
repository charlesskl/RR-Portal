# 利鸿出入库规则设计

## 目标

以真实文件 `东莞加工厂利鸿77794PCB主板出入明细(1).xlsx` 为准，修正利鸿部门的页面语义及 Excel 往返规则。

## 业务规则

- 利鸿页面只提供“领料”和“半成品出库”，不提供“成品入库”。
- 内部继续使用 `issue` 表示领料、`semi_finished` 表示半成品出库，以兼容已有数据。
- Excel 保持原始命名：`领料明细` 与 `半成品入仓明细`。
- 利鸿应存数等于领料总数减半成品出库总数。
- `半成品入仓明细` 的日期、送货单号、合同号、货号、品名/规格、数量和备注必须可以原样导入导出。
- 同一份数据经过“导入 -> 导出 -> 再导入”后不得产生重复记录。
- 鸿亚和其他部门现有规则不变。

## 数据设计

在 `records` 表增加可空字段 `contract_no`、`item_no`、`product_name`。新字段通过幂等迁移添加；旧记录保持可读。利鸿旧表导入时写入结构化字段，导出时优先使用结构化字段；历史记录缺少字段时使用现有默认值兼容导出。

## 页面与接口

- 利鸿录入类型删除 `finished`，将 `semi_finished` 显示为“半成品出库”。
- 后端拒绝利鸿新增 `finished` 记录。
- 利鸿汇总表显示“领料总数 / 半成品出库总数 / 应存数”。
- 利鸿 PCBA 导出仅包含 `总表`、`领料明细`、`半成品入仓明细`。

## 验证

- 静态页面测试确认利鸿类型选项和标签。
- API 测试确认利鸿拒绝成品入库。
- 导入导出测试确认真实字段、三张工作表及闭环防重复。
- 运行加工管理完整测试套件和 JavaScript 语法检查。
