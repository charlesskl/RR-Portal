# RR Portal Docker 操作手册

---

## 1. 启动整个 Portal

打开 CMD 或 PowerShell，执行以下命令：

```bat
cd "C:\Users\Administrator\Desktop\Project\RR portal"
docker compose up -d --build
```

等待所有服务构建完成后，即可访问。

### 检查服务状态

```bat
docker compose ps
```

确认所有服务状态为 `running` 或 `healthy`。

### 访问地址

| 页面 | 地址 |
|------|------|
| Portal 主页（控制台） | `http://localhost` |
| 工程啤办单系统 | `http://localhost/rr/` |
| Zuru MA 包装差价系统 | `http://localhost/zuru-ma/` |

> 局域网其他设备访问：将 `localhost` 替换为本机 IP（如 `http://192.168.2.151`）。
> 查看 IP 方法：CMD 中输入 `ipconfig`，找「以太网」下的 IPv4 地址。

### 关闭所有服务

```bat
docker compose down
```

## 2. 日常操作

### 每次开机后启动（已构建过，无代码改动）

```bat
cd "C:\Users\Administrator\Desktop\Project\RR portal"
docker compose up -d
```


```bat

```

## 3. 从 GitHub 拉取更新并重新部署

### GitHub 仓库信息

| 插件 | 本地路径 | GitHub 仓库 |
|------|----------|-------------|
| RR 工程啤办单 | `plugins\工程啤办单\` | `https://github.com/hufan4308-blip/工程啤办单.git` |

> 注意：目前只有 工程啤办单 有 GitHub 仓库。其他服务（如 zuru-ma、Portal 主页）暂无 Git 管理。

### 3.1 只更新 RR 工程啤办单（最常用）

```bat
:: 第一步：进入插件目录，拉取最新代码
cd "C:\Users\Administrator\Desktop\Project\RR portal\plugins\工程啤办单"
git pull origin main

:: 第二步：回到项目根目录，重新构建并部署（两个服务都会用到这份代码）
cd "C:\Users\Administrator\Desktop\Project\RR portal"
docker compose up -d --build rr-production plugin-rr-production
```

### 3.2 手动修改代码后重新部署

修改完代码后，重新构建并启动对应服务：

```bat
cd "C:\Users\Administrator\Desktop\Project\RR portal"

:: 只重建某个服务（不影响其他服务）
docker compose up -d --build rr-production
docker compose up -d --build zuru-ma
docker compose up -d --build plugin-rr-production
```

### 3.3 更新了前端页面（HTML/CSS/JS）

前端文件在 `frontend/` 目录下，Nginx 直接挂载了这个目录。

- 修改 `frontend/index.html` 或 `frontend/static/plugins/` 下的文件
- **刷新浏览器即可**，无需重启任何服务

### 3.4 修改了 Nginx 配置

```bat
docker compose restart nginx
```

### 3.5 修改了 docker-compose.yml

```bat
docker compose up -d
```

## 4. 添加新插件到某个部门

### 第一步：创建插件文件

将插件的前端页面放到：
```
frontend/static/plugins/<插件名>/
├── index.html
├── style.css
└── ...
```

如果插件有独立后端服务，在 `plugins/` 下创建：
```
plugins/<插件名>/
├── server.js
├── package.json
└── Dockerfile
```

### 第二步：配置路由（有后端服务时）

编辑 `docker-compose.yml`，添加新服务：
```yaml
my-service:
  build:
    context: ./plugins/my-plugin
    dockerfile: Dockerfile
  environment:
    - PORT=3000
  restart: unless-stopped
  networks:
    - platform-net
```

编辑 `nginx/nginx.conf`，添加路由：
```nginx
upstream my_service {
    server my-service:3000;
}

location /my-service/ {
    proxy_pass http://my_service/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

### 第三步：在 Portal 主页注册

编辑 `frontend/index.html`，在对应部门的卡片区域添加插件入口。

### 第四步：部署

```bat
cd "C:\Users\Administrator\Desktop\Project\RR portal"
docker compose up -d --build
```

## 5. 常见问题

### Q: 局域网其他设备无法访问？

1. 确保设备连在同一局域网
2. 用 `ipconfig` 查看正确的以太网 IP（不要用 198.18.x.x 的 VPN 地址）
3. 检查防火墙是否放行了端口 80：
   ```
   控制面板 → Windows Defender 防火墙 → 高级设置
   → 入站规则 → 新建规则 → 端口 → TCP 80 → 允许连接
   ```

### Q: Docker 服务启动失败？

```bat
docker compose logs <服务名>
```

### Q: 端口 80 被占用？

```bat
netstat -ano | findstr ":80 "
```
找到占用的 PID 后关闭对应程序，或修改 `docker-compose.yml` 中 nginx 的端口映射。
